# SkyMCP - Premium Weather Web App

A full-stack, natural language weather application powered by the **Model Context Protocol (MCP)** and a local **Ollama** model (`gemma4:e4b`). This application demonstrates how to bridge local LLMs with external APIs using standardized tool-calling protocols through a beautiful glassmorphism web interface.

## Architecture

The system follows a modular full-stack architecture:
- **Web Frontend (`frontend/`)**: A React/Vite application with premium CSS (glassmorphism, Framer Motion animations, Lucide icons) for interacting with the AI.
- **Web Backend (`main.py`)**: A FastAPI server bridging the frontend to the LLM. It routes user queries to Ollama and dynamically fetches available MCP tools.
- **MCP Weather Server (`mcp_weather_server.py`)**: A standalone FastAPI server acting as an MCP provider, exposing a `get_weather` tool securely connected to the OpenWeather API.

For a detailed technical breakdown and Mermaid diagram, refer to [architecture.md](./architecture.md).

## Prerequisites

- **[Node.js & npm](https://nodejs.org/)**: Required to run the React frontend.
- **[uv](https://docs.astral.sh/uv/)**: A fast Python package and environment manager.
- **[Ollama](https://ollama.com/)**: Required to run the local LLM.
- **`gemma4:e4b` Model**: Pull the model using `ollama pull gemma4:e4b`.
- **OpenWeather API Key**: Get a free key from [OpenWeatherMap](https://openweathermap.org/api).

## Installation

1. **Clone the repository**:
   ```bash
   git clone https://github.com/hellonihar/web_utility_to_check_weather_MCP_Local_LLM.git
   cd web_utility_to_check_weather_MCP_Local_LLM
   ```

2. **Setup environment variables**:
   Create a `.env` file in the root directory and add your OpenWeather API key:
   ```env
   OPENWEATHER_API_KEY=your_api_key_here
   ```

3. **Initialize the Python environment (Backend)**:
   ```bash
   uv venv
   # Windows: .venv\Scripts\activate
   # Unix: source .venv/bin/activate
   uv pip install mcp fastapi uvicorn ollama httpx pydantic requests python-dotenv
   ```

4. **Install Frontend Dependencies**:
   ```bash
   cd frontend
   npm install
   ```

## Usage

To run the application, you need to start three separate processes in three different terminal windows:

1. **Start the MCP Weather Server**:
   In your first terminal, launch the server that provides the weather tools (runs on port 8000):
   ```bash
   uvicorn mcp_weather_server:app --port 8000
   ```

2. **Start the Web Backend (FastAPI)**:
   In a second terminal, run the backend API layer (runs on port 8001):
   ```bash
   python main.py
   ```

3. **Start the React Frontend**:
   In a third terminal, start the Vite development server (usually runs on port 5173):
   ```bash
   cd frontend
   npm run dev
   ```

**Access the App**: Open your browser and navigate to `http://localhost:5173`. 
Select a city from the dropdown and click "Show Weather" to see the dynamic response generated by your local LLM utilizing the MCP weather tool!

## Project Structure

- `frontend/`: The React + Vite frontend application.
- `main.py`: The FastAPI backend bridging the frontend and the LLM agent.
- `mcp_weather_server.py`: The FastAPI server implementing the OpenWeather tool via MCP.
- `llm_mcp_client.py`: An alternative CLI-based interactive client for testing.
- `architecture.md`: Detailed architectural documentation.
